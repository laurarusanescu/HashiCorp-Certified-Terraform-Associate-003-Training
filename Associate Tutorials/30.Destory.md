## Overview

## 1Ô∏è‚É£ Destroy Infrastructure

### Two destruction methods
| Method | Description |
|--------|--------------|
| **Remove resources from config + apply** | Deletes specific resources selectively |
| **Run destroy operation** | Deletes **all** resources managed by the workspace |

---

### Using the HCP Terraform UI
1. Go to your workspace (e.g., **learn-hcp-terraform-run-triggers**).  
2. Click **Settings ‚Üí Destruction and Deletion**.

This page gives two options:
- **Queue destroy plan** ‚Üí Destroys all infrastructure managed by this workspace.  
- **Delete from HCP Terraform** ‚Üí Deletes the workspace without touching infrastructure.

‚ö†Ô∏è **Warning:** Deleting a workspace **does not destroy** its infrastructure ‚Äî resources (e.g., EC2 instances) remain active in the provider.

3. Click **Queue destroy plan**.  
4. Type your workspace name to confirm.  
5. Click **Confirm & apply**, then **Confirm plan**.

Repeat these steps for each workspace (e.g., `learn-hcp-terraform`).

## 2Ô∏è‚É£ Delete Workspace
Destroying infrastructure removes resources but keeps the workspace.  
To delete the workspace entirely (including variables and configuration):

1. Go to **Settings ‚Üí Destruction and Deletion**.  
2. Under **Delete Workspace**, click **Delete from HCP Terraform**.  
3. Confirm the workspace name when prompted.  

üí° **Tip:** Keep the workspace if you plan to redeploy resources later ‚Äî variables and configuration will remain intact.

## 3Ô∏è‚É£ Delete Project and Variable Set
If you want to completely remove your HCP Terraform project:

1. From your **organization overview**, click **Projects**.  
2. Choose your **Learn Terraform** project.  
3. Click **Settings ‚Üí Delete project**.  
4. Enter the project name and confirm deletion.

Deleting the project also deletes any **scoped variable sets** (e.g., AWS credentials).

‚ö†Ô∏è **Tip:** Keep the project if you plan to use it for future tutorials or additional workspaces.
